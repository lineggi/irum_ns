# 이룸 논슬립 (irumns.com) — 제품 요구사항 정의서 (PRD)

> 인터넷으로 논슬립을 구매하려는 **개인 구매자**를 위한, 저렴하고 쉬운 논슬립 판매·안내 사이트.
> 실제 결제는 네이버 스마트스토어(`smartstore.naver.com/anjeon_nonslip`)에서 이루어지고,
> 웹사이트는 **제품 카탈로그 + 시공 가이드 + 신뢰 구축 + 구매 유입** 역할을 한다.

---

## 1. 개요 (Overview)

### 1.1 배경
- 기존에는 FileZilla로 HTML을 카페24에 수동 업로드하던 정적 사이트를 운영.
- 이번에 GitHub + 카페24 FTP 자동 배포 파이프라인을 갖췄고(=코드 푸시 시 자동 반영), 이 위에서 UI/UX를 전면 개편한다.
- 논슬립은 "계단 미끄럼 방지"라는 명확한 검색 수요가 있는 제품이지만, 개인 구매자는 **"어떤 제품을 얼마나 사야 하는지, 혼자 시공이 가능한지"**를 몰라 구매를 망설인다.

### 1.2 문제 정의 (Problem)
1. 개인 구매자가 제품 종류(논슬립/마감재/굽도리/기타)와 규격을 이해하기 어렵다.
2. "혼자 시공할 수 있을까?"에 대한 확신이 없어 이탈한다.
3. 실제 구매는 네이버 스마트스토어에서 일어나는데, 그곳까지 자연스럽게 연결되는 퍼널이 없다.

### 1.3 해결 방향 (Solution)
- 제품을 **용도·상황 중심**으로 쉽게 소개하고(“우리 집 계단엔 뭐가 맞지?”),
- **DIY 시공 가이드**로 “혼자서도 가능하다”는 확신을 주고,
- **시공 사례**로 신뢰를 쌓은 뒤,
- 모든 경로가 **스마트스토어 “구매하기”**로 자연스럽게 수렴하도록 설계한다.

### 1.4 범위 (Scope)
- **In scope:** 정적 마케팅/카탈로그 사이트, 제품 소개, 시공 가이드, 시공 사례, 회사 소개, 문의(폼+채널), 스마트스토어 구매 연동, SEO, 반응형.
- **Out of scope (1차):** 자체 장바구니·결제·회원가입·주문/배송 관리(→ 스마트스토어가 담당). 향후 확장 후보.

## 2. 목표 & 성공 지표 (Goals & KPIs)

| 목표 | 지표 (KPI) | 측정 방법 |
| --- | --- | --- |
| 스마트스토어로 구매 유입 | “구매하기” 클릭 수 / 방문자 (전환율) | GA4 이벤트 `click_buy` |
| 검색 유입 확대 | 네이버/구글 자연 유입 세션 수 | GA4 + 네이버 애널리틱스 |

| 모바일 경험 | 모바일 세션 비중, LCP < 2.5s | GA4 + Lighthouse |

> 1차 핵심 KPI: **“구매하기” 전환율**과 **검색 유입량**.

## 3. 타깃 사용자 & 페르소나

- **P1. 집수리 개인 (주력)** — 아파트/주택 계단·현관·화장실 미끄럼이 걱정. 저렴하게, 혼자 시공하고 싶음. “계단 논슬립 셀프시공”으로 검색.
- **P2. 소상공인/자영업자** — 상가 계단·경사로 안전 문제(낙상 민원). 소량이지만 확실한 제품과 시공법을 원함.
- **P3. 시니어 가정 보호자** — 부모님 낙상 예방. 사용 편의·안전성·A/S를 중시.
- 공통점: **모바일**로, **네이버 검색**을 통해 유입될 가능성이 높다 → 모바일 우선 + 네이버 SEO 필수.

## 4. 핵심 사용자 시나리오 (User Flows)

1. **탐색→구매(주 퍼널):** 네이버 검색 → 홈/제품 → 제품 상세(규격·가격·용도) → “구매하기”(스마트스토어) → 결제.
2. **확신→구매:** 제품 상세 → “혼자 시공하기” 가이드 열람 → “이 제품으로 구매하기” → 스마트스토어.
3. **신뢰→문의:** 시공 사례 열람 → 문의하기(견적/규격 상담) → 전화/톡톡/폼.
4. **재방문:** 시공 중 궁금 → 시공 가이드 재방문(북마크/홈 화면 추가).

각 페이지에는 **상시 “구매하기” CTA**와 **문의 채널**이 노출되어 어느 경로에서든 전환 가능해야 한다.

## 5. 정보 구조 (IA) / 사이트맵

```

│   ├─ 논슬립      (/products/nonslip)

## 6. 기능 요구사항 (페이지별)

### 6.1 홈 (/)
- 히어로: 한 줄 가치제안(“미끄럼 걱정 없는 계단, 직접 만드세요”) + 대표 이미지 + **[구매하기]/[제품 보기]** CTA.
- 카테고리 4종 바로가기 카드(논슬립/마감재/굽도리/기타).
- “혼자 시공하기” 진입 배너(가이드로 유도).
- 대표 시공사례 3~4개, 신뢰 요소(누적 판매/후기/스마트스토어 평점 배지, 링크).
- 문의 채널(전화/톡톡) 노출.

### 6.2 제품소개 (/products 및 카테고리)
- 카테고리별 제품 **목록**: 썸네일, 제품명, 규격(폭×길이/두께), 용도 태그, 가격 참고, **[구매하기]**.
- 필터/정렬(용도: 실내/실외/계단/경사로, 재질 등) — 1차엔 태그 기반 단순 필터.
- **제품 상세**:
  - 이미지 갤러리(기존 이미지 그대로 활용), 규격표, 특징, 적용 위치, 색상/옵션.
  - **“이 제품 구매하기”** → 스마트스토어 **해당 상품 딥링크**(가능하면 상품별 URL, 없으면 스토어 홈).
  - “필요 수량 계산” 도우미(예: 계단 단수 → 필요 길이/개수 추정) — 개인 구매자 배려(2차 가능).
  - 관련 시공 가이드/사례 링크.

### 6.3 시공가이드 (/guide) — **차별화 핵심**
- “혼자 시공하는 방법”을 단계별로: 준비물 → 바닥 청소/재단 → 부착/마감 → 주의사항 → FAQ.
- 사진/도식 + (가능하면) 영상(유튜브 임베드).
- 상황별 가이드(계단, 현관, 경사로, 욕실 등)와 제품 추천 연결 → 구매 CTA.
- SEO 롱테일 타깃(“계단 논슬립 셀프시공”, “논슬립 붙이는 법”).

### 6.4 시공사례 (/cases)
- 현장 사진(전/후), 사용 제품, 장소 유형, 간단 후기.
- 사례 → 사용 제품 상세/구매로 연결.

### 6.5 회사소개 (/about)
- 회사 소개, 취급 품목, 강점(가격/품질/A/S), 위치(지도), 사업자 정보.

### 6.6 문의하기 (/contact)
- 문의 폼(이름/연락처/문의유형/내용) → **이메일 전송**(서버리스 폼 서비스).
- 전화, 네이버 톡톡/카카오 채널, 스마트스토어 문의 링크.
- 사업자 정보/개인정보 수집 동의 문구.

### 6.7 구매하기
- 전 페이지 공통 CTA. 새 탭으로 스마트스토어 이동. 클릭 시 GA4 이벤트 발생(전환 측정).

### 6.8 공통 요소
- 헤더(내비 + 구매 CTA), 푸터(사업자 정보/연락처/링크), 모바일 하단 고정 액션바, 플로팅 “맨 위로”, 404 페이지.

## 7. 비기능 요구사항 (Non-functional)

- **반응형:** 모바일 우선(360px~), 태블릿, 데스크톱. 터치 타깃 ≥ 44px.
- **성능:** LCP < 2.5s, 이미지 최적화(WebP + 적정 사이즈 + lazy-load), 폰트 최소화. Lighthouse 90+ 목표.
- **SEO:** 시맨틱 HTML, `<title>`/메타 디스크립션/OG 태그, 구조화 데이터(JSON-LD: `Product`, `LocalBusiness`, `BreadcrumbList`), `sitemap.xml`, `robots.txt`, **네이버 서치어드바이저/구글 서치콘솔 등록**.
- **접근성(A11y):** 색 대비 WCAG AA, 이미지 alt, 키보드 내비, 폼 레이블.
- **브라우저:** 최신 크롬/사파리/삼성인터넷/엣지, iOS·Android 모바일.
- **분석:** GA4 + 네이버 애널리틱스. 핵심 이벤트: `click_buy`, `submit_inquiry`, `click_call`, `view_guide`.
- **개인정보:** 문의 폼 최소 수집 + 동의 문구, 사업자 정보 표기(전자상거래 표시 의무).

## 8. 기술 아키텍처 & 스택 (추천)

### 8.1 핵심 원칙
> **결제·주문은 스마트스토어에 위임 → 우리 사이트는 정적(Static)으로 충분.**
> 정적 사이트는 (1) 카페24 호스팅과 호환, (2) 빠르고 SEO에 유리, (3) 운영이 단순, (4) 이미 만든 FTP 자동배포와 맞물린다.

### 8.2 추천안 A (권장): 정적 사이트 제너레이터 — **Astro** (또는 Eleventy)
- **이유:** 최종 산출물이 **순수 HTML**(카페24 정적 호스팅에 그대로 업로드) → SEO·성능 최상. 컴포넌트/레이아웃 재사용, **콘텐츠 컬렉션**으로 제품·사례·가이드를 데이터로 관리, 이미지 최적화 내장.
- **콘텐츠 관리:** 제품/사례/가이드를 `src/content/`의 **Markdown + 프론트매터** 또는 **JSON**으로 관리 → 비개발자도 항목 추가가 비교적 쉬움.
- **빌드/배포:** GitHub Actions에서 `astro build` → `dist/`를 **카페24 `/www/`로 FTP 업로드**(이미 구성한 `deploy.yml` 방식 재사용, `local-dir: ./dist/`로 조정).

### 8.3 추천안 B (초경량 무빌드): 순수 HTML/CSS/JS 멀티페이지
- **이유:** 빌드 단계 없이 파일을 그대로 FTP 업로드. 가장 단순.
- **한계:** 헤더/푸터 중복, 제품이 늘면 유지보수 부담 → 제품 수가 적고 변경이 드물 때 적합.
- 데이터는 `products.json` 등으로 분리하고 약간의 JS로 목록 렌더 가능(단, 클라이언트 렌더는 SEO에 불리하므로 상세 페이지는 정적 HTML 권장).

> **결론:** 제품/사례/가이드가 계속 늘어날 것을 감안하면 **추천안 A(Astro)**. “일단 최소로” 원하면 B로 시작해 A로 이관 가능.

### 8.4 곁다리 서비스(정적 사이트 보완)
- **문의 폼:** Web3Forms / Formspree / Getform 중 하나(무료 티어, 폼 → 이메일). 백엔드 불필요.
- **채팅/상담:** 네이버 톡톡, 카카오톡 채널 링크.
- **지도:** 네이버/카카오 지도 임베드.
- **영상:** 유튜브 임베드(시공 가이드).

### 8.5 데이터 모델 (예시 스키마)
```jsonc
// product
{
  "slug": "nonslip-alu-50",
  "category": "nonslip",           // nonslip | finish | skirting | etc
  "name": "알루미늄 논슬립 50mm",
  "images": ["/images/products/alu-50-1.jpg"],   // 기존 이미지 경로 그대로 사용
  "specs": { "width_mm": 50, "length_mm": 1000, "material": "알루미늄" },
  "use": ["실내", "계단"],          // 필터 태그
  "priceNote": "1m 기준 000원~",
  "buyUrl": "https://smartstore.naver.com/anjeon_nonslip/products/000",  // 상품 딥링크
  "relatedGuides": ["stair-diy"]
}
// case (시공사례) / guide (시공가이드) 도 유사한 프론트매터 구조
```

### 8.6 배포/운영 (기존 파이프라인 재활용)
- GitHub 저장소 `lineggi/irum_ns` + 브랜치 자동 배포.
- `deploy.yml`(Cafe24 FTP) 재사용: 빌드 산출물(`dist/`)을 `/www/`로 업로드. 시크릿 `FTP_HOST/USERNAME/PASSWORD`는 이미 설정됨(스크린샷 확인).
- 이미지: **기존 이미지 그대로 이관**(원본 파일명/폴더 유지) 후 WebP 최적화는 점진 적용.

## 10. 개발 로드맵 (마일스톤)

**M0. 기획·디자인 확정 (0.5주)** — IA/와이어프레임, 디자인 톤(색/폰트), 데이터 스키마 확정.
